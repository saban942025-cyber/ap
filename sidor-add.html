הקמת לקוח (Customers)

כדי שהלקוח ייכנס לאפליקציה ויזהה את הפרויקטים שלו, הדוקומנט ב-Firestore חייב להיראות כך.
* חשוב: שם הדוקומנט חייב להיות ה-UID מ-Authentication.

שדות מפתח:

  • projects: מערך של אובייקטים. קריטי להצגת הצ'אט.
  • tier: מאפשר להציג מחירים שונים (Gold/Silver).
  • preferences: הגדרות התראה אישיות.
Firestore JSON
"name": "בר אוריל קבלנות", "email": "bar@saban.co.il", "phone": "050-1234567", "customerNumber": "601235", "tier": "GOLD", // דירוג לקוח "avatarUrl": "https://...", "projects": [ { "projectId": "proj-asher-levy", "name": "פרויקט אשר לוי", "address": "ההסתדרות 29, רעננה", "coords": { "lat": 32.18, "lon": 34.86 } } ], "preferences": { "notifyOn": ["whatsapp", "push"], "deliveryWindow": "07:00-16:00" }

הקמת עובד (Users)

עובדים נשמרים בקולקציית users. השדה הקריטי כאן הוא role שקובע לאיזה אפליקציה תהיה לו גישה.

Role: "worker"

גישה לאפליקציית הנהג/מחסנאי. קבלת משימות והתראות.

Role: "admin"

גישה מלאה למערכת הניהול (Saban 360) ולדוחות.

Firestore JSON
"name": "יוסי הנהג", "email": "yossi@saban.co.il", "phone": "052-9999999", "role": "worker", // admin / worker / driver "vehicleNumber": "77-856-12", "vehicleType": "משאית מנוף 15 טון", "active": true, "lastActive": "2023-10-27T08:30:00Z"

קטלוג חכם (Products)

המוצרים מוצגים באפליקציית הלקוח. השדות המיוחדים כאן מאפשרים לבוט ולמחשבונים באפליקציה לעבוד.

לוגיקת מחשבון:

השדה calculator מאפשר לאפליקציה לחשב כמויות ללקוח בזמן אמת.
לדוגמה: עבור גבס, הנוסחה היא area / 3.12.

Firestore JSON
{ "name": "לוח גבס עמיד מים (ירוק)", "sku": "10-500", "category": "גבס", "price": "₪42.00", "image": "https://...", "spec": "120x260 ס\"מ, עובי 12.5 מ\"מ", "stock": 150, "description": "מתאים לחדרים רטובים.", // שדות מתקדמים לבוט ולאפליקציה: "technicalData": { "application": "קירות פנים רטובים", "dryingTime": "24 שעות לשפכטל" }, "calculator": { "unit": "m2", "formula": "(area / 3.12) * 1.1" // כולל 10% פחת }, "relatedProducts": ["sku-10-501", "sku-20-100"] }
